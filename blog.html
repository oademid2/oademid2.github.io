Polygon is the scaling solution Ethereum needs

What it is and why I’m bullish | 01/25/2022

What is Polygon?

Polygon is a Layer 2 chain for creating Ethereum compatible networks. In simple terms, it allows you to execute smart contracts on its chain that can be transferred between Ethereum, and supports the same for independent chains. 

Why I’m bullish?

Polygon is Scalable.

Being a layer 2 network, Polygon solves Ethereum’s biggest issue of scalability. As activity on Ethereum increases, it faces high congestion. This has resulted in slower transactions and higher gas fees. As a layer 2 solution, Polygon is able to process transactions of Ethereum’s mainnet while still holding the integrity of Ethereum’s block chain.

Polygon supports a multichain future.

Polygon’s vision is to provide connectivity between different blockchains and be Ethereum’s internet of blockchains. In essence, it enables blockchains to easily bridge with Ethereum. This is promising because the future of blockchain will likely be multichain and connectivity between different blockchains will improve the user experience. What’s more, Ethereum is one of the most utilized blockchains by developers so strategically there’s a lot of opportunity in being the gateway for other blockchains to integrate with it.

Vitalik says the future is multichain:

Polygon emphasizes value of Ethereum.

Lastly, outside of its scalability issues, Ethereum already provides significant functionality.  Unlike some layer 1 solutions to Ethereum’s problems (i.e. Solana, Cardano)* Polygon builds on Ethereum instead of in place of Ethereum. It’s EVM compatible which means that smart contracts deployed on the Ethereum network can be easily deployed on Polygon’s blockchain thus low switching costs. Ethereum is a largely used and trusted network with lots of strong functionalities, it makes sense to leverage this. Betting on Polygon is also betting on Ethereum.

To summarize -- not only does polygon have improved scalability, its low switching costs and EVM compatibility reduces barriers for adoption.

*I own a bit of Cardano and previously held Solana and currently considering re-holding

Below is a bit more detail on Polygon’s functionality:

How does Polygon work?

Polygon uses an adaptation ofEthereum’s Plasma’s framework which allows transactions to occur off the main Ethereum network and later transferred. Very simply, transactions will occur on Polygon and be bundled together as one transaction when ready to be carried over to the main Ethereum network. These individual transactions are verified on Polygon as they occur and when returned to Ethereum as a batch, are then verified by Ethereum.

Polygon aims to provide several options for scaling solutions. It’s primary solution is the PoS chain which is a permissionless side chain that uses a proof-of-stake consensus mechanism with it’s own validators, but also relies on Ethereum’s security for validator staking and check points. 

They’ve recently implemented XX from their initial roadmap and are looking to develop additional scaling solutions such as xx.

The 4 layers of Polygon

Overall, polygon can be broken into 4 layers -- two mandatory and two optional layers.

The Ethereum layer, the first optional layer, allows Polygon chains to use Ethereum as it’s base layer for things like staking, dispute resolution , messaging and a few other functionalities. The security layer, the second layer, is also optional. It allows chains to add a validation as a service or use Ethereum as a security layer. The third layer is the polygon networks layer where core functions such as transaction collation, local consensus and block production occur. The last layer is the ethereum layer, which is mandatory, and interprets and executes transactions that are included in Polygon’s chains

The Polygon bridge

The bridge is a key part of Polygon. It is how assets are migrated between layer 2 and Ethereum. There are two implementations offered -- Plasma bridge and PoS bridge.

The Plasma bridge uses a mechanism called Fraud proofs for the main chain, Ethereum, to validate the assets being transferred from Polygon. A challenge period of 7 days is required when withdrawing funds which means it will take up to 7 days for funds to be withdrawn unto Ethereum.

With the PoS bridge Proof of stake based assets provides PoS security. Since this security is accepted by users and developers it does not use fraudproffs and allows for a faster exit with one checkpoint interval.

In conclusion

Polygon solves Ethereum’s most pressing problem of scalability without directly competing against Ethereum.
